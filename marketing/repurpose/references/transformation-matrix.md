# Transformation Matrix

Complete source × target rules. Each cell tells you what to keep, cut, add, and the target length.

---

## LinkedIn Post → Everything

### → X/Twitter Thread
```
KEEP: Core insight, key data points, personal story elements
CUT: Context paragraphs, LinkedIn-specific CTAs ("comment below"), blank line spacing
ADD: Thread opener tweet (strongest claim), "🧵" indicator, self-reply with link
REWRITE: Hook (shorter, punchier), CTA (retweet/follow instead of comment)
LENGTH: 5-10 tweets, 200-270 chars each
STRUCTURE: Tweet 1 = hook/claim, Tweets 2-N = evidence, Last tweet = CTA + open question
```

### → Single Tweet
```
KEEP: The single most quotable line OR the contrarian take
CUT: Everything else. A tweet is one idea, not a summary.
ADD: Nothing. Simplicity is the point.
REWRITE: Must work with zero context. Reader hasn't seen the LinkedIn post.
LENGTH: 200-270 characters
RULE: If you can't pick one line, the post didn't have a strong enough core insight.
```

### → Email
```
KEEP: Story, insight, CTA
CUT: LinkedIn-specific hooks (ultra-short openers feel weird in email), engagement bait
ADD: Personal opening ("I've been thinking about..."), P.S. with bonus value or second CTA
REWRITE: Subject line (from the hook, but email-style: curiosity or personal), tone (more intimate)
LENGTH: 150-300 words body + subject (30-50 chars) + P.S. (1-2 sentences)
STRUCTURE: Subject → 1-line hook → story → insight → CTA → P.S.
```

### → Carousel (LinkedIn/Instagram)
```
KEEP: Framework, steps, key data
CUT: Narrative prose (carousels are visual, not literary)
ADD: Title slide (hook as headline), summary slide (key takeaway), CTA slide
REWRITE: Each paragraph → 1 slide with 30-50 words max. Use bold for key phrases.
LENGTH: 8-12 slides, 30-50 words per slide
STRUCTURE: Slide 1 = hook, Slides 2-N = one point each, Last slide = CTA/summary
```

### → Short Video Script
```
KEEP: Personal story, key insight, 1 specific example
CUT: Multiple points (video = ONE idea), data-heavy sections (boring on camera)
ADD: Visual cues ("[show screen]", "[cut to...]"), call to action spoken naturally
REWRITE: Present tense, conversational. Written to be spoken, not read.
LENGTH: 60-90 seconds (150-225 words)
STRUCTURE: Hook (3 sec) → Context (10 sec) → Story (30 sec) → Insight (15 sec) → CTA (5 sec)
```

## Blog Post → Everything

### → LinkedIn Series (3-5 posts)
```
KEEP: Each major section becomes its own post
CUT: Intro/conclusion (each post needs its own), transitional paragraphs
ADD: Unique hook per post, standalone context, individual CTA
REWRITE: Each post must work independently. Reader shouldn't need parts 1-2 to understand part 3.
LENGTH: 1,300-2,000 chars per post
RULE: If a section can't stand alone, merge it with another or cut it.
```

### → X/Twitter Thread
```
KEEP: Main thesis + 3-5 strongest supporting points
CUT: Nuance, caveats, lengthy examples (save for blog). Usually 70% of the blog.
ADD: Thread opener with bold claim, numbering ("1/"), final tweet with link to full post
REWRITE: Every sentence must earn 280 characters. Kill filler ruthlessly.
LENGTH: 6-12 tweets
RULE: Thread should make reader think "I need to read the full thing" — it's a trailer, not the movie.
```

### → Email Sequence
```
KEEP: Each section = 1 email. Key insight + 1 example per email.
CUT: Academic depth (save for blog). Each email is a conversation, not a chapter.
ADD: Personal angle per email, bridge to next email ("tomorrow I'll share..."), P.S.
REWRITE: Blog is authoritative. Email is personal. "I" and "you" in every paragraph.
LENGTH: 3-5 emails, 200-300 words each
STRUCTURE: Each email follows: hook → story/example → one takeaway → CTA → P.S.
```

## Video/Podcast → Everything

### → Blog Post
```
KEEP: Key insights, stories, frameworks mentioned
CUT: Filler ("um", "so", tangents), conversational loops, repeated points
ADD: Structure (headers, sections), links/references mentioned verbally, images/charts
REWRITE: Spoken language → written language. Clean up grammar, add paragraph breaks.
LENGTH: 1,500-3,000 words depending on episode length
PROCESS: Transcribe → Extract key moments → Outline → Rewrite in blog voice
```

### → LinkedIn Post
```
KEEP: The single best story or insight from the episode
CUT: Everything else. 30 min of content → 1 post about 1 thing.
ADD: Context (listener doesn't know the episode), hook, CTA
REWRITE: Conversational spoken style → LinkedIn personal-professional style
LENGTH: 1,300-2,000 chars
RULE: Pick the moment that made you think "that's the clip"
```

### → Short Clips
```
KEEP: 3-5 strongest moments (15-60 seconds each)
CUT: Setup, context, tangents around the moment
ADD: Caption/subtitle, hook text overlay in first 2 seconds
LENGTH: 15-60 seconds per clip
SELECTION CRITERIA: Does this clip make sense without the full episode?
```

## Case Study → Everything

### → LinkedIn Story Post
```
KEEP: Customer's problem, what they tried, your solution, the result (with numbers)
CUT: Company background, technical implementation details
ADD: Hook from the result ("From 2 users to 1,000 in 6 weeks"), your reflection
REWRITE: Third person → first person narrative. "We worked with..." → "6 weeks ago, [name] had a problem."
LENGTH: 1,300-2,000 chars
```

### → Ad Copy
```
KEEP: Problem + result + specific numbers
CUT: Everything between problem and result (the ad IS the compression)
ADD: CTA, urgency if applicable
REWRITE: UGC style. "I was spending 8h/month on reports. Now it takes 15 minutes."
LENGTH: 80-125 chars primary text + 25-40 chars headline
```

### → Email
```
KEEP: Full story arc (problem → struggle → solution → result)
CUT: Technical details unless audience is technical
ADD: "Could this be you?" bridge, CTA to demo/trial, P.S. with second proof point
LENGTH: 300-500 words
```
